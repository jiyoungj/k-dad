Database Description

1. Data collection

This section describes the recording procedures, experimental design, and equipment used during the data collection process.

1.1. Devices used

Articulatory data: Carstens AG501
Acoustic data: Microphone & TASCAM

1.2. Experiment setup

• Sound-attenuated room in Hanyang Institute for Phonetics and Cognitive Sciences of Language
• TASCAM HC-P2 digital recorder and a SHURE KSN44 condenser microphone
• Eight sensor coils attached (tongue tip, body, dorsum, lower/upper incisors, lower/upper lips, nasion)

5. Technical specifications

This section explains how recordings and annotation files are organized and named in the database, data processing procedures, sampling rate, and related technical details.

5.1. Labeling conventions

All files follow the naming convention: [TaskType]_[Label]_[Sentence]_[Rate]_[Dialect][Age][Sex][ID]_[Repetition]
The fields are defined as follows:

  • TaskType: PA (Text Passage) or SY (Isolated Syllables)
  • Label: the passage identifier (A–G) for passage recordings, or the syllable set identifier for syllable recordings
  • Sentence: the specific sentence within a passage, or the specific syllable within a set
  • Rate: F (Fast), N (Normal), or S (Slow)
  • Dialect: dialect region code, e.g. SEK (Seoul), NKK (North Gyeongsang), SJK (South Jeonnam), etc.
  • Age: speaker's age group (20–60)
  • Sex: M (Male) or F (Female)
  • ID: speaker number within the group (01–30)
  • Repetition: R01 (first repetition) or R02 (second repetition)

Each recording consists of three files sharing the same base filename:

  • wav audio file,
  • mat articulatory data file (MATLAB format), and
  • TextGrid text annotation file.

TextGrids contain three interval tiers:

  1. 발화 — sentence-level segmentation in Hangul orthography;
  2. Utterance — the same sentences in Google Korean Romanization; and
  3. 단어 / Word — word-level segmentation with each label giving the Hangul form and its romanization separated by a forward slash (e.g., 북풍과/bugpunggwa).

5.2. Data processing details

Articulatory data were recorded using the Carstens AG501 Electromagnetic Articulography (EMA) system. Sensors were attached to the following articulators: tongue tip, tongue body, tongue dorsum, upper lip, lower lip, and lower gum. Additionally, sensors were placed on the nasion and upper gum as fixed reference points for head movement correction.

Recordings were made as continuous sweeps of complete text passages and complete syllable sets. Each sweep was subsequently segmented into individual files at the sentence level for passage recordings and at the individual syllable level for syllable recordings. Following segmentation, TextGrid annotation files were created for each recording through hand-labeling in Praat, providing sentence-level transcriptions in both Hangul and Google Korean Romanization, as well as word-level segmentation.

Raw sensor position data were processed in MView. Head movement correction was performed using the nasion and upper incisor/gum reference sensors to remove the effects of head movement during recording. Following head correction, sensor coordinates were rotated into the occlusal plane via bite plane correction, which was recorded from each speaker prior to the main recording session. The .mat files distributed with the database contain the fully head-corrected and bite-plane-corrected sensor trajectories.

Acoustic and articulatory signals were recorded simultaneously on a single system.

5.3. Sampling rate

Acoustic recordings are sampled at 48,000 Hz at 16-bit depth and stored in WAV format.
Articulatory data are sampled at 1,250 Hz and stored in MATLAB (.mat) format.
